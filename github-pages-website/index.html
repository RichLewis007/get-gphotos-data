get-gphotos-data

Python desktop GUI application to access and view Google Photos data via their Library API

Python PySide6 Google Photos API OAuth 2.0

Google has ended support for the Google Photos Library API features which allow users to write programs to read or list their existing photos and videos.

As of March 31, 2025, Google deprecated the photoslibrary.readonly scope that provided access to all photos in a user's library. The Library API now only supports accessing photos and videos that were uploaded by your application (photoslibrary.readonly.appcreateddata), which means this program cannot function as intended.

This project is deprecated and no longer maintained.

For accessing photos from Google Photos, you would need to use the Google Photos Picker API, which requires users to manually select photos through a web interface rather than programmatically listing all photos.

Quick Start

Installation

# Clone the repository git clone https://github.com/YOUR_USERNAME/Google-Photos-Library-API.git cd Google-Photos-Library-API # Install dependencies uv sync --dev # Run the application uv run get-gphotos-data

Authentication Setup

To use this application, you need to set up OAuth 2.0 credentials with Google:

  1. Create a Google Cloud Project
  2. Enable the Google Photos Library API
  3. Create OAuth 2.0 credentials (Desktop app type)
  4. Download the credentials JSON file
  5. Place it in the project directory or select it when prompted

See the README for detailed instructions.

Note: As of March 31, 2025, the Google Photos Library API only provides access to media items and albums uploaded/created by your application. To access existing Google Photos, use the Google Photos Picker API instead.

Requirements

  • Python 3.13.11
  • uv (Python package manager)
  • Google Cloud Project with Google Photos Library API enabled
  • OAuth 2.0 credentials

Usage

  1. Authenticate: Click "Authenticate" button in the Google Photos tab
  2. Refresh Data: Click "Refresh Data" to fetch your Google Photos data
  3. Browse: View media items, albums, and shared albums in the respective tabs
  4. Details: Click any item to see full JSON details
